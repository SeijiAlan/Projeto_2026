# 📘 Versionamento e Controle de Código com Git e GitHub

## 📌 Sobre o Projeto

Este repositório foi desenvolvido como parte de uma atividade prática acadêmica com o objetivo de aplicar conceitos de versionamento de código utilizando Git e GitHub, simulando um fluxo de desenvolvimento profissional.

O projeto consiste na implementação de um sistema simples em Python, evoluído por meio da criação de novas versões organizadas em branches distintas, preservando a estabilidade da branch principal.

---

## 🎯 Objetivo

Aplicar na prática:

- Clonagem de repositórios (`git clone`)
- Inicialização de repositório local (`git init`)
- Navegação em diretórios via terminal
- Verificação de status (`git status`)
- Criação e gerenciamento de branches
- Versionamento de arquivos (`git add`, `git commit`)
- Envio de alterações para repositório remoto (`git push`)
- Alternância segura entre branches (`git switch`)

---

## 🔄 Estratégia de Versionamento

Foi adotado um fluxo baseado em boas práticas:

- `main` como branch estável
- `feature/versao2` para desenvolvimento isolado de nova funcionalidade
- Commits descritivos
- Verificação de diretório limpo antes de alternar branches

Esse modelo simula um ambiente real de desenvolvimento colaborativo.

---

## 🧠 Competências Desenvolvidas

- Organização de código por versão
- Controle de histórico de alterações
- Separação entre ambiente estável e desenvolvimento
- Uso consciente de branches
- Mentalidade de fluxo profissional de versionamento

---

- Python 3

## 📚 Contexto

Projeto desenvolvido para fins acadêmicos, com foco na consolidação de práticas fundamentais de versionamento e organização de projetos de software.
